Network Concepts

HTTP

HTTP(HyperText Transfer Protocol)는 웹 브라우저와 서버가 데이터를 주고받기 위한 통신 규약입니다. 사용자가 웹사이트에 접속하면 브라우저는 HTTP 요청(Request)을 서버에 보내고, 서버는 HTML, CSS, 이미지 등의 데이터를 응답(Response)합니다. 현재 대부분의 웹사이트는 보안 기능이 추가된 HTTPS 방식을 사용합니다.

DNS

DNS(Domain Name System)는 사람이 읽기 쉬운 도메인 주소를 실제 서버 IP 주소로 변환하는 시스템입니다. 예를 들어 seongmin6112.shop 같은 주소를 입력하면 DNS가 해당 서버의 IP 주소를 찾아 브라우저가 올바른 서버에 접속할 수 있도록 도와줍니다.

TCP/IP

TCP/IP는 인터넷 통신의 기본 구조입니다. IP는 데이터가 목적지까지 이동할 수 있도록 주소를 담당하고, TCP는 데이터가 순서대로 안전하게 전달되도록 관리합니다. 대부분의 인터넷 서비스는 TCP/IP 기반으로 동작합니다.

UDP

UDP(User Datagram Protocol)는 빠른 속도를 우선하는 통신 방식입니다. TCP와 달리 연결 과정을 단순화하여 속도는 빠르지만, 데이터 손실 여부를 직접 확인하지 않습니다. 실시간 스트리밍, 온라인 게임, TFTP 같은 서비스에서 자주 사용됩니다.

TFTP

TFTP(Trivial File Transfer Protocol)는 UDP 기반의 간단한 파일 전송 프로토콜입니다. 구조가 단순하여 네트워크 장비 설정 파일 전송, 부팅 파일 전송 등에 사용됩니다. 복잡한 인증 기능은 없지만 빠르고 가벼운 특징을 가지고 있습니다.

GitHub Pages

GitHub Pages는 GitHub 저장소의 HTML, CSS, JavaScript 파일을 웹사이트 형태로 배포할 수 있는 서비스입니다. 정적 웹사이트를 무료로 호스팅할 수 있으며, 커스텀 도메인 연결도 지원합니다.
