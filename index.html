NETWORK WORLD

인터넷과 웹 시스템은 다양한 네트워크 프로토콜과 구조를 기반으로 동작합니다. HTTP, DNS, TCP/IP 같은 기술들은 웹 브라우저와 서버가 서로 연결되고 데이터를 주고받을 수 있도록 도와줍니다.

Network Concepts

HTTP

HTTP는 웹 브라우저와 서버 간 데이터를 주고받기 위한 통신 규약입니다. 사용자가 웹사이트에 접속하면 브라우저는 서버에 요청(Request)을 보내고, 서버는 HTML, CSS, 이미지 같은 데이터를 응답(Response)합니다. 현재 대부분의 웹사이트는 암호화 기능이 추가된 HTTPS 방식을 사용합니다.

DNS

DNS는 사람이 읽기 쉬운 도메인 주소를 실제 서버 IP 주소로 변환하는 시스템입니다. 사용자가 도메인을 입력하면 DNS 서버가 해당 주소에 연결된 IP를 찾아 브라우저가 올바른 서버에 접속할 수 있도록 도와줍니다.

TCP/IP

TCP/IP는 인터넷 통신의 핵심 구조입니다. IP는 데이터가 목적지까지 이동하도록 주소를 담당하며, TCP는 데이터가 순서대로 안전하게 도착하도록 관리합니다. 대부분의 인터넷 서비스는 TCP/IP 기반으로 동작합니다.

Protocols

UDP

UDP는 빠른 속도를 우선하는 통신 방식입니다. TCP보다 구조가 단순하며 연결 과정을 최소화하여 빠른 데이터 전송이 가능합니다. 온라인 게임, 실시간 스트리밍, 음성 통신 등에 자주 사용됩니다.

TFTP

TFTP는 UDP 기반의 간단한 파일 전송 프로토콜입니다. 구조가 단순하여 네트워크 장비 설정 파일 전송, 부팅 파일 다운로드 등에 사용됩니다. 복잡한 인증 기능은 없지만 빠르고 가벼운 특징을 가지고 있습니다.

HTTPS

HTTPS는 기존 HTTP에 보안 기능을 추가한 프로토콜입니다. SSL/TLS 암호화를 통해 사용자의 데이터가 안전하게 전송되도록 보호합니다. 로그인 정보, 결제 정보 같은 중요한 데이터 보호에 사용됩니다.

Web Systems

GitHub Pages

GitHub Pages는 HTML, CSS, JavaScript 파일을 웹사이트 형태로 배포할 수 있는 서비스입니다. 정적 웹사이트를 무료로 호스팅할 수 있으며, 커스텀 도메인 연결도 지원합니다.

Web Browser

웹 브라우저는 사용자가 인터넷에 접속할 수 있도록 도와주는 프로그램입니다. 브라우저는 서버와 HTTP/HTTPS 통신을 수행하며, 전달받은 HTML과 CSS를 해석하여 화면에 출력합니다.

Server

서버는 클라이언트 요청을 처리하고 데이터를 제공하는 시스템입니다. 웹 서버는 사용자의 요청에 따라 HTML 파일, 이미지, 데이터 등을 전달하며 인터넷 서비스의 핵심 역할을 수행합니다.
